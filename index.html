About

This assignment focuses on understanding fundamental HTML elements and document structure. It explores the purpose and usage of the pre tag for displaying preformatted text, ensuring proper spacing and line breaks are preserved. Additionally, the assignment covers the CSS box model, which defines how elements are structured and spaced on a webpage.

Another key aspect is comparing browser default styles with custom styles. Browsers apply built-in styles to HTML elements by default, but developers can override these styles using CSS. By examining these differences, we gain insight into how styles affect webpage appearance and behavior.

The pre (preformatted text) tag preserves spaces, line breaks, and indentation exactly as written in the HTML, making it useful for displaying code or structured text.

The code tag is used to indicate inline or block-level code snippets, applying a monospace font for readability. It does not preserve formatting like pre, so it is often used inside pre for proper code display.

Box Model

This image visually represents the CSS Box Model, showing how margin, border, and padding work together. The outer dark box represents margin, the black outline is the border, and the inner beige box demonstrates padding around the content.

  • Margin: Space outside the border that pushes elements away
  • Border: Defines the element's edge
  • Padding: Creating space inside the border
